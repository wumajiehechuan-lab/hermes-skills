# 从概念到画布：Lean Canvas + VPC 生成器

## 核心理念

> **别写科幻小说。用一页纸把脑洞变成可验证的假设。**

从粗略概念到商业方案，最忌讳直接写几十页BP。正确路径是：
1. 先花15分钟填一张精益画布 → 找到漏洞
2. 针对卡住的格子，用VPC深度推演 → 验证PMF
3. 最后才考虑MVP和验证

**这个 skill 只负责前两步。** 让你在 1 小时内从"模糊感觉"到"可验证的一页纸方案"。

## 第一步：理解概念 & 联网背景调查

### 做什么

拿到用户输入后，先做两件事：

**① 理解概念边界：** 把用户的几句话/几个关键词，拆解为可填画的要素。需要追问时使用 `clarify` 工具。

**② 可选联网搜索：** 如果用户的概念涉及具体行业/市场（例如"做一款AI英语口语陪练"、"面向班主任的资料平台"），用联网搜索收集基础信息。使用以下工具：

| 工具 | 用途 |
|------|------|
| `web_search(query, limit=5)` | 搜索发现来源，支持 site:domain、引号精确匹配 |
| `web_extract(urls)` | 已知 URL 时提取全文（包括 PDF） |
| `browser_navigate(url)` | 上述两者返回空时的降级方案（JS 渲染的竞品官网、Product Hunt 页面） |
| 可选：Tavily | 已配置。做竞品调研/定价对比时优先使用，一次调用拿 `answer`（AI 摘要）+ 结果 |

搜索方向（搜完即止，不要过度搜索）：
- 这个领域的主要竞品有哪些？最简单的替代方案是什么？
- 目标人群的消费能力和付费习惯
- 市场是否有类似产品，定价多少
- 技术可行性（API成本估算）

> **搜索策略：** 至少尝试 3 种不同关键词组合再判断"没找到"。一手来源优先：官方定价页 > 原创评测 > 聚合帖。查不到的数据直接标 🔴 纯猜测，绝不编造。

> **判断标准：** 用户概念中的"假设"有公开数据可验证吗？有则搜，无则跳过。搜索结果直接填进 Lean Canvas 的可信度标注。

## 第二步：生成精益画布（Lean Canvas）

### 填充顺序和逻辑

严格按照以下**商业逻辑顺序**填充，**不是按格子编号顺序**：

| 顺序 | 格子 | 要填充的内容 |
|------|------|------------|
| ① | **问题 (Problem)** | 目标客户的 1-3 个最大痛点。标注哪些是"必须有"（紧急），哪些是"可以有"（锦上添花） |
| ② | **客户细分 (Customer Segments)** | 谁会买这个？早期尝鲜者画像是什么？（年龄、身份、使用场景、消费能力） |
| ③ | **独特价值主张 (UVP)** | 一句话解释：为什么你与众不同、值得关注？格式：`为[谁]解决[什么问题]，通过[什么方式]` |
| ④ | **解决方案 (Solution)** | 针对上面的痛点，MVP 阶段的 1-3 个核心功能。不是完整功能列表，是"最少需要具备什么才能解决痛点" |
| ⑤ | **渠道 (Channels)** | 如何触达目标用户？列出 2-3 个具体渠道（如小红书、知乎、Product Hunt、SEO、微信群等） |
| ⑥ | **收入来源 (Revenue Streams)** | 怎么收钱？订阅/买断/按量/免费+增值？定价区间？（标注：这是纯猜测，需要验证） |
| ⑦ | **成本结构 (Cost Structure)** | 最大的几笔开销？考虑：API算力、服务器、营销获客、支付手续费、人力时间成本 |
| ⑧ | **关键指标 (Key Metrics)** | 怎么证明产品在往对的方向走？列出 1-3 个北极星指标（如：首周注册数、付费转化率、留存率） |
| ⑨ | **门槛优势 (Unfair Advantage)** | 别人抄你有什么是他拿不走的？没有就空着，标注"暂缺——这是正常状态" |

### 重要规则

1. **每个格子标注可信度：** 🟢 有依据（来自联网搜索或用户明确描述）/ 🟡 合理推断 / 🔴 纯猜测
2. **留白允许：** 门槛优势（⑨）填不出就空着，标注"这是正常的，建议先做出来再说"
3. **区分"功能假设"和"价值假设"：** 在解决方案（④）中标注哪些是"我觉得用户会喜欢"（功能假设），哪些是"用户明确需要"（价值假设随搜索验证）

### 呈现格式示例

提交给用户时用清晰的段落+列表格式：

```
## 精益画布：[项目名称]

### ① 问题
1. 🟢 [痛点描述] — 来源：用户描述/联网搜索
2. 🟡 [痛点描述] — 来源：合理推断
3. 🔴 [痛点描述] — 来源：猜测

### ② 客户细分
[描述]

### ③ 独特价值主张
> 一句话价值主张

### ④ 解决方案
1. 🟡 [功能] — 价值假设（需要验证）
2. 🟡 [功能] — 功能假设（用户可能不需要）

### ⑤ 渠道
1. [渠道1]
2. [渠道2]

### ⑥ 收入来源
[描述] — 🔴 纯猜测，需验证

### ⑦ 成本结构
[描述]

### ⑧ 关键指标
[描述]

```

## 第三步：推演价值主张画布（VPC）

### 什么时候做

用户修改完 Lean Canvas 后。**至少 "问题" 和 "客户细分" 两个格子是清楚的**，否则 VPC 推演不扎实。

### 填充逻辑

VPC 是精益画布中**问题(①) + 客户细分(②) + 解决方案(④)** 三个格子的深度拆解。

**右侧 — 客户画像（Customer Profile）：**

| 要素 | 说明 |
|------|------|
| **客户任务 (Jobs)** | 用户想要完成什么任务？功能任务 / 社交任务 / 情感任务 |
| **痛点 (Pains)** | 过程中什么让他们烦恼、浪费时间、花钱、焦虑？ |
| **收益 (Gains)** | 什么会让他们开心、省钱、省时间、有成就感？超出期望的"爽点" |

**左侧 — 价值地图（Value Map）：**

| 要素 | 说明 |
|------|------|
| **产品与服务** | 你的产品具体提供了什么？ |
| **痛点消除器 (Pain Relievers)** | 每个功能具体消除了哪个痛点？一对一映射 |
| **收益创造器 (Gain Creators)** | 每个功能创造了什么超出预期的收益？ |

### 关键检查：齿轮是否咬合

```
每个"痛点" → 有没有对应的"痛点消除器"？
每个"收益" → 有没有对应的"收益创造器"？
每个"产品功能" → 至少对应一个"痛点"或"收益"？
```

- 有"痛点"没有对应的消除器 → 产品功能缺失
- 有"产品功能"没有对应的痛点 → "自嗨功能"，可以砍掉

### 呈现格式

```markdown
## 价值主张画布：[项目名称]

### 客户画像（右侧）

**客户要完成的任务（Jobs）：**
- 🟡 [任务1] — 功能任务
- 🟢 [任务2] — 社交任务

**痛点（Pains）：**
1. [痛点1]
2. [痛点2]

**收益（Gains）：**
1. [收益1]
2. [收益2]

### 价值地图（左侧）

**产品与服务：**
- [功能1] — [功能2] — [功能3]

**痛点消除器：**
| 痛点 | 消除器 | 匹配度 |
|------|--------|-------|
| 痛点1 | 功能1 | ✅ 直接对应 |
| 痛点3 | （无对应） | ❌ 待补充 |

**收益创造器：**
| 收益 | 创造器 | 匹配度 |
|------|--------|-------|
| 收益1 | 功能3 | ✅ 直接对应 |

### 齿轮咬合检查结果

- ✅ 痛点消除器覆盖：X/3
- ✅ 收益创造器覆盖：X/2
- ⚠️ 无对应痛点的功能：[功能列表] — 考虑砍掉或降级
- ❌ 未覆盖的痛点：[痛点列表] — 考虑补充功能
```

### 提交用户优化

生成完成后，**明确提问：**
> 这是基于你确认的 Lean Canvas 推演的 VPC。请看看：
> 1. 客户的任务/痛点/收益——有遗漏或偏差吗？
> 2. 齿轮咬合检查中的 ⚠️ 和 ❌ — 你打算怎么处理？
> 3. 整体感觉这个方案"咬合"了吗？
>
> 直接说需要改的地方，我优化后输出最终文档。

**等待用户反馈。**

**依赖：** 只需 `pip install markdown`（已安装）。WeasyPrint 和 Chrome 都是按需使用的引擎，不强制。

## 注意事项 / Pitfalls

1. **不要在用户没确认 Step 1 的情况下强行进入 Step 2。** 每一步都必须等待用户反馈。如果用户一次性要求都做完，可以在第一步完成后自动进入第二步，但每步仍需汇报结果并等待反馈。

2. **不要假装自己有数据。** 所有格子都要标注可信度（🟢🟡🔴）。没有搜索验证过的内容就是 🔴 纯猜测，要有勇气标注。

3. **门槛优势填不出来时，明确告诉用户"这是正常的"。** 不要编造"技术壁垒"、"数据积累"这类没有的东西。很多成功项目早期也没有门槛优势。

4. **输出路径确认：** 确保 `E:/work/hermes/concept-to-canvas-output/` 目录存在，不存在则创建。

5. **与 biz-canvas skill 的关系：** 本 skill 是"生成方案"，biz-canvas 是"分析已有方案"。如果用户后续有了具体竞品需要分析，推荐他切换到 biz-canvas。

6. **不要过度追问：** 如果用户只说了一句"帮我做个AI填色页"，直接生成画布，不要追问10个问题。填不出的格子标注 🔴 留空即可，用户会自己修正。

7. **中文优先：** 画布内容是中文的，因为用户的输出和反馈都是中文。

8. **PDF 生成是自动的：** 脚本自动检测可用引擎（WeasyPrint → Chrome headless），一行命令出 PDF，不需要手动干预。如果真到了两个引擎都不可用的极端情况，先交付 MD 并备注原因。
