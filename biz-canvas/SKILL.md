# 商业模式五维分析法

## 核心理念

> **九分靠营销，一分靠技术。**

看一个项目时，多看它的**流量来源（营销）**，少看它的**界面功能（技术）**。很多看起来"技术厉害"的产品，真正的壁垒可能在别处；很多看起来"没什么技术含量"的生意，赚钱的秘密可能在流量和组织上。

这套框架不是为了写研究报告，而是为了**快速判断一个项目值不值得做、能不能抄、天花板在哪**。

---

## 第一步：并行信息收集

使用子Agent并行搜索来收集五个维度的原始信息。建议的分工：

- **子 Agent 1 — 需求与变现**：定价策略、收费模式（订阅/按量/买断/广告/抽成）、竞品定价对比、用户评价和投诉分析（痛点是什么、愿意付多少）、毛利率估算
- **子 Agent 2 — 流量与增长**：获客渠道和冷启动路径、SEO 数据和关键词覆盖、社交媒体存在和活跃度、联盟/推荐计划、广告投放 ROI
- **子 Agent 3 — 壁垒与天花板**：技术实现方案评估（API 调用还是独立算法）、竞品护城河分析、TAM/SAM/SOM 市场规模数据、大厂动态（是否有平台整合风险）、生命周期判断

**子Agent 联网工具使用指南**（直接写入每个子Agent的prompt中）：

> 你需要联网获取信息。使用以下工具：
> - **web_search**：用于搜索发现信息来源，获取摘要和关键词结果
> - **web_extract**：当已知具体URL时，用于从页面定向提取内容
> - 搜索策略：先用web_search发现信息来源和线索，找到具体URL后用web_extract深入提取
> - 多次搜索、多个关键词组合，不要只搜一次就放弃
> - 一手来源优于二手来源：官方博客 > 权威媒体原创报道 > 转载/聚合
> - 对于 Indie Hackers 项目，优先抓取产品页面、创始人文稿、公开收入数据

prompt要描述目标（"获取""调研""了解"），不要用暗示具体手段的动词（"搜索""爬取"），让子Agent自主判断最佳获取方式。

### 信息充分性自检

搜索完成后检查：
- 五个维度中，哪个维度的信息最薄弱？补搜
- 定价数据和获客渠道两大项必须覆盖——这是判断商业本质的核心输入
- 信息不够就再补搜，不要凑合

## 🔍 核心维度一：需求与痛点（Problem & Solution）

任何长久的生意，底层都是在解决一个真实存在的需求。

### 需要回答的问题

**谁在用？**
- 目标用户的画像是什么（年龄、身份、消费能力）？
- 他们在什么场景下想起这个产品？
- 这个产品是给使用者本人，还是使用者以外的人付费的？

**他们为什么不用现成的免费替代品？**
- 免费的方案缺什么？是体验差、不精准、还是没覆盖这个细分场景？
- 用户之前尝试过什么替代方案？为什么那些方案没解决问题？

**止痛药还是维他命？**
- **止痛药**：用户有明确的痛苦，不解决会持续难受 → 付费意愿强
- **维他命**：用户觉得"有了更好"，但没有了也活得下去 → 付费意愿弱，需要靠营销持续提醒
- 判断方法：如果明天这个产品消失，用户会主动去找替代品吗？

## 💰 核心维度二：商业变现与成本（Monetization & Cost）

### 收费模式（Revenue Model）

一个生意怎么收钱，直接决定了它的毛利率、获客成本和用户生命周期。常见的微 SaaS 收费模式：

| 模式 | 优点 | 缺点 | 适合场景 |
|------|------|------|---------|
| **订阅制（月/年）** | 收入可预测，LTV 高 | 获客成本回收期长，需要持续提供价值 | 工具类、SaaS |
| **按量付费/积分制** | 低门槛试用，用户按使用付费 | 收入波动大，大户难预测 | AI 生成类（按张收费） |
| **买断制** | 一次性利润高 | 无持续收入，版本升级难推广 | 模板、数字素材 |
| **免费+广告** | 流量大，零付费门槛 | CPM 低，需要巨大流量 | 内容站、工具站 |
| **交易抽成** | 收入跟平台交易量绑定 | 需要双边网络效应 | 市场平台类 |

**关键分析**：这个项目是单一收入来源，还是多个收入引擎？每个引擎的占比和毛利是多少？

### 成本结构（Cost Structure）

AI 微 SaaS 的典型成本结构：

- **API 算力成本**：调用 LLM / 图像模型的费用。这是可变成本，与使用量成正比
- **服务器与域名**：固定成本，通常很低
- **支付处理费**：约 2.9% + $0.30/笔
- **内容审核成本**：对面向儿童或 UGC 的产品来说是隐性但重要的成本
- **营销获客成本**：大部分微 SaaS 的最大开销
- **人力成本**：单人创业时是自己的时间机会成本

**毛利率估算**：`（收入 - 可变成本）/ 收入`。AI 微 SaaS 的毛利率通常很高（70%+），前提是单次 API 调用的成本控制得当。

## 🚀 核心维度三：流量与增长（Traffic & Marketing）

这是独立开发者最容易失败的环节——产品做好了，没人知道。

### 冷启动与获客渠道（Acquisition）

**第一批用户是怎么来的？**
- 社交媒体（Reddit / Twitter / TikTok / Pinterest / Instagram）
- Product Hunt / Hacker News 发布
- 社区分享（相关论坛、群组、邮件列表）
- 付费广告（Facebook Ads / Google Ads / Pinterest Ads）
- SEO / 内容营销（博客、引导页、免费工具引流）
- 联盟/推荐计划

**核心追问**：每个渠道的获客成本（CAC）是多少？哪个渠道的 ROI 最高？

### SEO 潜力判断

- 用户会不会在搜索引擎上搜索这类关键词？
- 搜索量多少？竞争难度（Keyword Difficulty）高吗？
- 能不能通过程序化 SEO（批量生成着陆页）拦截长尾流量？

### 用户传播力（Virality）

- 用户会自发分享这个产品吗？分享的动机是什么？
- 有没有内置的传播机制（生成的内容自带品牌水印、社交分享按钮）？

## 🛡️ 核心维度四：技术壁垒与护城河（Moat & Defense）

评估这个项目是不是"套壳"，会不会明天就被大厂或抄袭者一波流带走。

### 技术门槛评估

- 它是独立开发的算法，还是调用的公开 API？
- 核心功能需要多少行代码实现？几天能复制？
- 有没有独特的数据积累、模型微调、或专利？

### 真正的护城河（如果不是技术）

对微 SaaS 来说，技术壁垒往往不是真正的壁垒。真正的护城河通常来自：

| 护城河类型 | 强度 | 说明 |
|-----------|------|------|
| **品牌认知** | 中强 | 用户心智一旦建立，后来者要花几倍的代价才能撬动 |
| **SEO 排名** | 中 | 先发优势 + 内容积累，后来者需要时间和资源追赶 |
| **用户数据/网络效应** | 强 | 用户越多产品越好用（如 Canva 的模板库） |
| **迁移成本** | 中强 | 用户已经存了大量内容在这个平台上，迁移麻烦 |
| **生态集成** | 中 | 跟其他工具深度绑定（如 Shopify 插件生态） |
| **许可证/合规** | 弱 | 某些行业需要资质认证 |
| **专利/IP** | 弱（对微 SaaS） | 申请成本高，维权难 |

### 核心检验

> **如果我今天花三天抄一个一模一样的，我能抢走他的客户吗？如果不能，为什么？**

如果答案是"能，因为没区别"——这个项目长期存活的概率很低。
如果答案是"不能，因为他已经有 X 万的 SEO 页面和用户评论积累"——这是真实的护城河。

## 📈 核心维度五：天花板与生命周期（Ceiling & Lifecycle）

看这个项目的长远价值，它是能长大的"树"，还是只能赚零花钱的"灌木"？

### 市场规模（TAM / SAM / SOM）

- **TAM（Total Addressable Market）**：这个品类的全球市场总规模
- **SAM（Serviceable Addressable Market）**：你能服务到的市场（考虑地理、语言、支付方式限制）
- **SOM（Serviceable Obtainable Market）**：你短期内实际能拿下的份额

### 天花板判断

- 这是**利基市场（Niche Market）**还是**大众市场（Mass Market）**？
- 利基市场：上限可能不高（月入 $1-10万），但竞争低、利润高，足够养活一个单人团队
- 大众市场：天花板高但竞争激烈，需要更大的团队和资本

### 大厂碾压风险

- 如果 ChatGPT / Google / Adobe / Canva 这类大厂下场做同样的功能，你会不会直接死掉？
- 大厂通常不会做特别垂直的小功能（不够大，不值得专门做个产品线），但可能通过平台整合来覆盖这个场景
- 防御策略：**做深不做宽**——在垂直场景里做到最好，让大厂觉得"不值得专门做"或者"做了也打不过"

### 生命周期判断

- 这个需求是**一阵风**（如 2023 年的 AI 头像生成），还是**长期存在的需求**（如填色/教育/打印）？
- 长期需求 + 持续运营 = 可以做几年以上的"长期饭票"
- 短期风潮 + 流量驱动 = 适合快进快出，不要投入大量基建

### 延伸空间

- 这个产品能往哪些方向长？
- 向上：加功能、提单价（从工具变平台）
- 向旁：覆盖相邻人群/场景（从填色页到绘本、活动书）
- 向深：做社区/生态（从工具变 UGC 平台）

## 📋 商业分析速查清单

看到一个新项目时，按这个顺序快速过一遍：

| 维度 | 核心提问 | 危险信号 |
|------|---------|---------|
| **需求** | 谁在用？为什么不用免费替代品？ | "有了更好"的维他命，用户迁移成本为零 |
| **钱** | 怎么收钱？扣掉 API/算力成本后毛利多少？ | 算力成本 > 月费，越卖越亏 |
| **流量** | 用户怎么知道这个产品的？渠道可持续吗？ | 全靠烧广告，没有自然流量引擎 |
| **竞争** | 三天抄一个能抢走客户吗？ | 答案是"能"，且你没有其他壁垒 |
| **未来** | 这是长期需求还是一阵风？大厂下场会死吗？ | 需求是短命的，或者大厂一句话就能覆盖 |

> **记住一句话：九分靠营销，一分靠技术。**
