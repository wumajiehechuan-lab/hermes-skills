# 商业概念打磨 Workout

## 核心理念

> **商业方案不是写出来的，是"练"出来的。**

这个 skill 把精益创业三部曲做成**多轮对话**。不是一次灌给你几十个问题，而是每次聊 1-2 个格子，你回答完、我帮你整理、保存当前版本，下一轮继续深化。

**输出：** 每次对话都会在 `E:/work/hermes/business-concept-workout-output/` 下保存版本文件，你随时可以叫停。

**与 biz-canvas 的关系：**
- `biz-canvas` = 侦探显微镜，拆解**已有项目**
- `business-concept-workout` = 乐高说明书，从零生成**新方案**
- 可以先用 workout 打磨出方案，然后用 biz-canvas 验证这个方案行不行

---

## 触发条件

当用户抛出以下信号时激活本 skill：

> 「我有个想法……」/「你说这个事儿能不能做」/「我突然想到一个点子」/「帮我梳理一下这个方向」/「我有个粗略概念」

或者用户表达了类似"我脑子里只有几个词/一句话，不知道怎么展开"的意图。

---

## 总流程：3 层深度

整个流程分三个深度层次，**不要一次性全部抛给用户**，而是从第 1 层开始，逐层递进：

```
第 1 层：RAW INTAKE —— 理解原始概念（1-2 轮）
     ↓
第 2 层：LEAN CANVAS —— 9 格骨架（3-5 轮，每轮 1-2 格）
     ↓  ← 用户随时可以叫停，保存当前版本
第 3 层：WEAK SPOT DEEP DIVE —— 弱项深挖（按需触发）
     ├─ 问题/客户格子很虚 → 价值主张画布 (VPC)
     ├─ 渠道格子很虚 → Fake Door 实验设计
     ├─ 竞争看不懂 → 竞品速查
     └─ 整体信心不足 → 精益创业假说验证设计
```

---

## 🌟 第 1 层：RAW INTAKE（原始概念接引）

### 核心任务
了解用户的原始概念，确认方向，为精益画布做准备。

### Agent 行动指南

**第一步：** 当用户说"我有个想法"或类似内容时，用开放式提问接住：

> 好的，你脑子里这个想法是什么？哪怕只有几个词、一句话都可以。先说出来，我们慢慢展开。

**第二步（可选）：** 如果用户表达得过于模糊（比如只有一个词），可以追加 1-2 个引导性问题，**但不要超过 2 个**：

- 「这个想法主要是解决谁的什么问题？」
- 「你为什么会觉得这是个机会？」

**第三步：** 大致理解后，给出确认总结（1-2 句话），然后自然过渡到精益画布：

> 好的，我理解你的概念方向了。接下来我们来做精益画布——把想法塞进 9 个格子里，看看骨架。我们从第一个问题开始：**你的目标用户有哪些最大的痛点？**

---

## 🏗️ 第 2 层：LEAN CANVAS（精益画布骨架）

### 核心规则（非常重要）

- **每轮只填 1-2 个格子**，不要一次性抛出所有问题
- **严格按商业逻辑顺序**（见下方），不要打乱
- 如果用户对某个格子答不上来，标记为「待验证假设」，跳到下一个
- 每轮结束时，自动保存当前版本（v1.md, v2.md...）

### 填充顺序（每次 1-2 个问题）

| 轮次 | 格子 | 引导问题 |
|------|------|---------|
| 1 | ①问题 (Problem) + ②客户细分 | "目标用户的**3 个最大痛点**是什么？这些痛点是谁最有感觉？" |
| 2 | ③独特价值主张 (UVP) | "用一句大白话说——凭什么用户要来关注你？你跟现在的解决方案有什么不同？" |
| 3 | ④解决方案 (Solution) | "针对刚才说的痛点，你的产品**核心功能**是什么？最少要做什么才能解决痛点的 80%？" |
| 4 | ⑤渠道 (Channels) | "这些人平时在哪里出没？你怎么让他们知道你？社媒、SEO、还是社区口碑？" |
| 5 | ⑥收入来源 + ⑦成本结构 | "怎么收钱？做出来最大的几笔开销是什么？（API 费、服务器、还是你自己的时间？）" |
| 6 | ⑧关键指标 + ⑨门槛优势 | "怎么知道做对了？上线第一周看到什么数字你就算成功？别人想抄你，什么东西他拿不走？如果没有就先空着。" |

### 每轮的交互格式

```
[确认总结]
「好的，我来梳理一下你刚才说的……（1-2 句话概括用户回答）」

[编号格子内容]
在本轮更新的格子里把用户回答结构化填进去

[保存版本]
「当前进度：已完成 3/9 格 → 已保存至 v1.md」

[引导下一步]
「接下来我们看看怎么把产品送到用户手里——你最想通过什么方式让他们知道这个东西？」
```

## 🔍 第 3 层：WEAK SPOT DEEP DIVE（弱项深挖）

### 3A — 价值主张画布 (Value Proposition Canvas)

适用场景：用户对「问题」和「客户细分」格子没把握。

**用一张图引导用户：**

```
┌───────────────────┐      ┌─────────────────────────┐
│  你的产品          │      │  客户画像                │
│                   │      │                         │
│  Product & Service │      │  Jobs (他们要完成的任务)    │
│  （做什么）          │      │  1. ...                  │
│                   │  ←→  │  2. ...                  │
│  Pain Relievers   │      │                         │
│  （消除什么痛苦）      │      │  Pains (让他们烦的事)      │
│                   │      │  1. ...                  │
│  Gain Creators    │      │  2. ...                  │
│  （创造什么收益）      │      │                         │
│                   │      │  Gains (他们想要的爽点)     │
│                   │      │  1. ...                  │
│                   │      │  2. ...                  │
└───────────────────┘      └─────────────────────────┘
```

**引导逻辑（分 3 轮）：**

1. **Jobs + Pains**：「你的用户平时在这个场景下，具体要做什么事？过程中有什么让他们烦躁、浪费时间、或者觉得不痛快的地方？」
2. **Gains**：「如果这个场景能做到完美，他们最想要的「超出期待」的结果是什么？」
3. **Pain Relievers + Gain Creators**：「你的产品是如何解决上面那些痛点、触及那些爽点的？一五一十对号入座。」

**价值主张画布的核心检验：**
> 画完之后，对比左右两侧——每个 Pain 和 Gain 右边有没有对应的 Pain Reliever / Gain Creator？如果有 Pain 没有对应的产品功能，说明产品还没完全对准用户真实需求。

### 3B — Fake Door 实验设计

适用场景：用户对「渠道」格子没把握，或者想验证需求。

**自动输出 Fake Door 实验方案：**

```markdown
## Fake Door 实验方案

**假设验证：** [提取用户画布中最致命的假设]

**怎么做：**
1. 用 1 小时做一个 Landing Page（用 Carrd / 简单的 HTML 页面）
2. 页面内容：
   - 标题：［你的 UVP］
   - 正文：2-3 行描述解决了什么问题
   - CTA：「立即订阅—早鸟价 $XXX」
3. 点击 CTA → 弹出「Coming Soon，留下邮箱」
4. **投放渠道：** ［基于用户回答的渠道］
5. **成功标准：** 3 天内 >50 人留邮箱

**判断标准：**
- 0-10 人 → UVP 没打中，需要重新提炼价值主张
- 10-50 人 → 需求存在但不够强烈，或者渠道没选对
- 50+ 人 → 需求确认，可以考虑做 MVP
```

### 3C — 竞品速查（调用 web_search）

适用场景：用户担心竞争或不知道现有替代方案。

使用 sub-agent 或 web_search 做轻量竞品快速扫描：

```
搜索指令：
web_search("用户描述的领域 + 替代品/工具/方案")
web_search("细分场景 + best tool")
web_search("关键词 + review/alternatives")

整理格式：
| 竞品 | 模式 | 价格 | 差距（用户可以去抢的机会） |
|------|------|------|--------------------------|
| A    | 订阅 | $XX  | 没有 X 功能               |
| B    | 免费 | $0   | 体验差，用户抱怨 Y         |
| C    | ...  | ...  | ...                      |
```

**核心追问：**
> 用户现在用什么方案凑合？你要比这个「凑合方案」好多少才值得他们改变习惯？

### 3D — 精益创业假说验证设计

适用场景：整体画布填完了但信心不足，想设计最低成本的验证方案。

**自动输出：**

```markdown
## 最致命的假设（Riskiest Assumption）
［从画布中提取 1-2 个必须为真才能活下去的假设］

## MVP 方案（最小的验证，不是最小的产品）
- 最轻量验证：［1 小时的 Fake Door / 3 天的手动服务］
- 中等验证：［1 周的半自动化原型］
- 完整 MVP：［2 周的可交付产品］

## 验证周期
建议 7 天内完成一轮 Build → Measure → Learn：
| 第 1-2 天 | 第 3-5 天 | 第 6-7 天 |
|-----------|-----------|-----------|
| 上线最小验证 | 观察真实数据 | 决定 Pivot / Persevere |
```

## ⚠️ 避坑指南

1. **不要一次性抛出全部问题**。每次只问 1-2 个，保持对话感。用户答完一个再问下一个。
2. **不要替用户编答案**。如果用户答不上来，标记为「待验证」，不要自行脑补。
3. **不要用商业术语吓唬用户**。引导语用大白话——比如说「谁来买」而不是「目标客户画像」。
4. **不要等到全填完才保存**。每轮结束自动保存当前版本。
5. **区分「假设」和「事实」**。画布上写的是假设，不是答案。用 [假设] 标记所有未验证的内容。
6. **提前设定期望**。开场时可以告诉用户：「这是一个迭代过程，每次聊 1-2 个问题，几轮下来就有完整方案了。你随时可以叫停。」

- **plan：** 如果方案通过了评估，用户可以说「帮我写一个落地计划」，切换到 plan skill 做执行计划。

>
> *（用户回答）*
>
> **Agent：** 清楚了。帮你记下：
> 1. 每天为了找选题和素材花 2 小时以上
> 2. 作图效率太低，AI 工具出的图格式不对
> 3. 不知道什么内容类型容易火
>
> **还有第二个问题：这些痛点谁最有感觉？——是刚起步的萌新博主、还是已经有几千粉的腰部博主？**
>
> *（用户回答）*
>
> **Agent：** 好的，第一轮填了 2 格 → 已保存 v1。接下来我们聊聊一句话价值主张——**凭什么别的博主不用免费的 Canva 或者稿定设计，要来用你的东西？**
