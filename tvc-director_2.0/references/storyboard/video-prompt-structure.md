# Multi-Phase 视频提示词结构

> 视频提示词结构与语法、双图输入、与多宫格的对应关系。Phase 5 加载。

## 核心转换思路

把"是什么"变成"怎么变"。

| 维度 | 静态关键帧 | 视频提示词 |
|------|-----------|-----------|
| 时间 | 无 | 有时间码，精确到秒 |
| 主体 | 冻结状态/姿态 | 运动中的过程（动作+变化）|
| 镜头 | 固定景别+视角 | 运镜轨迹（推/拉/环绕/跟随）|
| 画面衔接 | 无 | 转场指示（硬切/溶解/匹配剪辑）|
| 光影 | 固定光源方向 | 光影随时间变化 |

## Multi-Phase 视频提示词结构

```
风格：[视觉风格] / [色彩基调] / [光影系统] / [约束条件] / 无背景音乐 产品@产品多视图图片 的广告

Phase 1 (0-Xs): [标题]
[景别+视角] [运镜描述]。[产品/主体状态变化]。[光影效果]。[功能揭示（如有）]。

Phase 2 (X-Ys): [标题]
[节奏变化描述]。[运镜描述]。[产品动态]。[光效变化]。

...

光影要求：[贯穿全片的光影系统描述]
```

每段视频独立编号：Phase 从 1 开始，秒数从 0 开始，不延续上一段。

## 双图输入

视频模型接收两张图片——多宫格分镜图（首帧）+ 产品多视图（产品锚定）。

- 风格声明末尾用 `产品@产品多视图图片 的广告` 引用产品多视图
- `(图1)(图2)` 参考图映射是**多宫格图片提示词**的语法，**不用于视频提示词**

## 音频铁律

- 视频模型默认生成 BGM，不写就会有
- 每段风格声明必须写 `无背景音乐`
- BGM 在后期作为单独音轨统一铺设

## 不写后期指令

以下属于后期工作，不写入视频提示词：Logo/品牌标识叠加、Slogan/文字渐显、调色指令、音效设计、字幕排版。

## Phase 与多宫格的对应关系

Phase 的画面顺序对应多宫格从格 1 到格 9 的顺序。一个 Phase 覆盖 1-3 个连续的格子——把这几格之间的运动和转场填充为连贯的镜头段落。一个 Phase 是一个连贯场景，不要在 Phase 内部描述不同场景之间的快速交叉剪辑。

## 逐镜描述公式（七要素）

```
[时间码][景别+视角] + [镜头运动] + [主体动作] + [环境/光效变化] + [角色对白(如有)] + [转场指示]
```

**写作要点**：动词为王，渐变优于突变，具体优于抽象（`旋转30°` 优于 `缓慢旋转`），因果链。

## 15 秒标准节奏编排

| 时段 | 秒数 | 功能 | 节奏 |
|------|------|------|------|
| **定场** | 0-3s | 建立场景/氛围 | 慢 |
| **发展** | 3-7s | 展开叙事/展示 | 中速 |
| **高潮** | 7-12s | 情绪/动作顶峰 | 快 |
| **收束** | 12-15s | 留白/余韵/CTA | 减速 |

## 跨段连续性

- 所有段共享相同风格声明
- 无背景音乐每段必写
- 角色外观通过同一资产图引用保持一致
- 前段末尾色调 ≈ 后段开头色调

### 段间独立性

每段 Phase 编号从 1 开始、秒数从 0s 开始。不要在本段描述中引用其他段已发生的事件。
